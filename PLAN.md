# خطة تنفيذ موقع "معمل الماسة للتحاليل الطبية" — عرض تفاعلي (Discovery & Proposal Website)

## 1. طبيعة المشروع
هذا ليس الموقع النهائي للمعمل، بل **صفحة عرض تفاعلية Single Page** تشرح تصور المشروع الرقمي الكامل
(الموقع + بوابة المريض + برنامج الموظفين + لوحة الإدارة + النظام الموحد) لصاحب المعمل، وتتيح له اتخاذ
قرار لكل قسم (اعتماد / تعديل / تأجيل / استبعاد) مع تدوين ملاحظاته، بحيث تتحول الصفحة نفسها إلى
"مرجع القرار الرسمي" (Master Decision Log) لبدء التنفيذ الفعلي لاحقًا.

## 2. المكدس التقني (Stack)
- **Vite + React 19 + TypeScript** — أساس المشروع (SPA بالكامل، بدون Backend).
- **Tailwind CSS v4** (عبر `@tailwindcss/vite`) — تصميم سريع ومتّسق، دعم RTL كامل.
- **Framer Motion** — حركات/انتقالات fluid (scroll reveal, hero parallax خفيف, tabs, accordions).
- **lucide-react** — أيقونات SVG متّسقة.
- **localStorage** — تخزين قرارات العميل وملاحظاته والاقتراحات المضافة (بدون Backend/DB).
- **GitHub Actions + GitHub Pages** — نشر تلقائي عند push إلى الفرع الرئيسي (build فقط، static).

لا يوجد Backend حقيقي، لا API مدفوعة، لا أسرار/مفاتيح. كل شيء static + client state.

## 3. البنية المعمارية للمحتوى
كل الأقسام الـ23+ مبنية من **بيانات نمطية موحّدة** (TypeScript content modules) بدلاً من صفحات منفصلة،
حتى يسهل الصيانة والتوسعة:

- `src/content/sections.ts` — تعريف كل قسم (id, عنوان, أيقونة, نوع المحتوى).
- `src/content/*.ts` — محتوى كل قسم (نصوص، تحديات، حلول، رحلة المريض، صلاحيات، تسعير مراحل...).
- `src/content/suggestions.ts` — الاقتراحات الإضافية الذكية (10–20) لكل قسم مرتبط.
- `src/types/decision.ts` — نوع قرار العميل: `approved | approved_with_changes | deferred | excluded`.

### مكوّن "قرار العميل" الموحّد (`<DecisionBlock sectionId=... />`)
يظهر في نهاية كل قسم رئيسي: 4 أزرار قرار + textarea ملاحظات + زر "اقتراحات إضافية" (accordion لكل
اقتراح بثلاث حالات: إضافة/تجاهل/ناقش لاحقًا). كل تغيير يُحفظ فورًا في `localStorage` عبر hook
`useDecisionsStore`. الافتراضي لأي قسم بدون تفاعل = "معتمد كما هو" (implicit).

### لوحة MASTER DECISION LOG
في نهاية الصفحة: تجميع تلقائي حي (state-driven) لكل القرارات + الاقتراحات المُضافة + الملاحظات، مع:
- طباعة العرض (`window.print` + CSS مخصص للطباعة)
- تصدير القرارات (JSON قابل للتنزيل)
- تصدير النطاق المعتمد (نص/Markdown قابل للتنزيل يلخّص ما تم اعتماده فقط)
- حفظ التقدم محليًا (تلقائي + رسالة تأكيد)

## 4. مراحل التنفيذ (تُنفَّذ بالتسلسل دون توقف)
1. **Bootstrap**: Vite + TS + Tailwind v4 + Framer Motion + lucide + هيكلة المجلدات + ESLint/oxlint.
2. **نظام القرار الأساسي**: types + store (localStorage) + DecisionBlock + SuggestionsBlock عام.
3. **الهيكل العام**: Layout (Header sticky + progress bar + RTL) + Footer + SEO/meta + favicon.
4. **الأقسام 1–7**: Hero، ملخص تنفيذي، التحديات، الحل المقترح، رحلة المريض، الاستعلام، بوابة المريض.
5. **الأقسام 8–14**: برنامج Windows، إدخال النتائج، كتالوج التحاليل، نقل قاعدة البيانات، حسابات المرضى،
   واتساب، الذكاء الاصطناعي.
6. **الأقسام 15–18**: لوحة الإدارة، الصلاحيات، الأمان، التقارير.
7. **الأقسام 19–23**: 3 اتجاهات تصميمية (Mockups)، مقارنة قبل/بعد، مراحل التنفيذ (Timeline)، FAQ، CTA ختامية.
8. **Master Decision Log + Footer الإلزامي بالروابط**.
9. **الصقل البصري**: موبايل أولاً، طباعة، `prefers-reduced-motion`، فحص تراكب أفقي، إمكانية وصول.
10. **إعداد GitHub Pages**: base path، GitHub Actions workflow، اختبار 404/refresh.
11. **فحص نهائي شامل**: build، lint، مراجعة تفاعلات، تحديث السجلات، commit + push متكرر بعد كل مرحلة.

## 5. معايير الجاهزية للإنتاج (Definition of Done)
- `npm run build` بدون أخطاء.
- `npm run lint` نظيف (أو تحذيرات مبررة فقط).
- لا Placeholders / Lorem Ipsum / نصوص إنجليزية غير ضرورية.
- عربي RTL كامل، لا overflow أفقي من 320px فأكثر.
- يعمل الحفظ عبر localStorage عبر إعادة تحميل الصفحة.
- GitHub Actions workflow جاهز للنشر على GitHub Pages مع base صحيح لاسم الريبو.
- الفوتر يحتوي على الروابط المطلوبة بالضبط.

## 6. الملفات المرافقة
- `IMPLEMENTATION_LOG.md` — سجل زمني لكل مرحلة، ما تم، مشاكل البناء وحلولها.
- `CONTENT_MAP.md` — خريطة كل قسم إلى ملف المحتوى ومكوّن العرض.
- `DECISION_LOG.md` — توثيق القرارات التصميمية/التقنية التي اتخذتها أثناء البناء (وليست قرارات العميل).
- `FEATURE_MATRIX.md` — جدول كل اقتراح إضافي: القيمة، التعقيد، الأولوية، المرحلة المقترحة، السبب، الفائدة.
